# v0 PMS Dashboard - Branch Strategy

## Working Branch: `main`

All development happens on **`main`** branch.

### Workflow

```bash
# 1. Make changes locally
git checkout main
# ... edit files ...

# 2. Commit changes
git commit -m "type: descriptive message"

# 3. Push to GitHub
git push origin main

# 4. Vercel automatically deploys when main is updated
```

### What's in main

✅ Landing page with Yagán branding + animations
✅ Login modal with Supabase authentication  
✅ PMS Dashboard with full hotel management
✅ Case studies (including Chile market content)
✅ i18n support with Spanish/English
✅ All bug fixes:
  - Canvas gradient error prevention
  - ReadableStream locked error fix
  - Hydration fixes

### Deployment

- **Branch**: main
- **Platform**: Vercel (automatic deployment on push)
- **Status**: Production ready
- **Domain**: www.yagan.app

### Important Notes

- Never work on feature branches, always commit to `main`
- Vercel watches `main` for automatic deployments
- All code must compile (run `npm run build` before pushing)
- Test locally with `npm run dev` before committing
